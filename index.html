DOCTYPE

É fundamental incluir o elemento <!DOCTYPE> no início de seus documentos HTML. Embora não seja um elemento HTML em si, sua presença orienta o navegador sobre como interpretar o documento. Isso assegura o funcionamento adequado e a melhor interpretação do conteúdo pelo navegador.

  1. <!DOCTYPE html>

Títulos Inteligentes

É recomendável atribuir um título significativo ao seu documento HTML. Utilize palavras-chave relevantes relacionadas ao conteúdo da página para facilitar sua localização através de buscas no navegador.

  1. <title>Título Inteligente</title>

Metatags

É essencial dominar o uso das meta tags, pois desempenham um papel muito importante na configuração das páginas. Elas servem como parâmetros de configuração, permitindo que você indique ao navegador como a página deve se comportar. As meta tags são úteis para especificar detalhes como a forma como a página será compartilhada, fornecer informações sobre o conteúdo para programas externos e mecanismos de busca, entre outras funcionalidades.

As três principais metatags para aprender sendo iniciante são:

  1. <meta charset=”UTF-8”> <!-- Serve para informar ao documento quais caracteres devem ser processados e exibidos, impedindo que as letras e caracteres especiais fiquem bagunçados ou inexistentes. -->
  3. <meta name=”viewport” content=”width=device-width, initial-scale=1.0”> <!-- Serve para identificar que esse documento vai ser adaptado para diversos dispositivos, como computadores, tablets, smartphones, televisores, entre outros. -->
  5. <meta name=”description” content=”Descrição”> <!-- Aqui você coloca uma breve descrição atraente para o conteúdo de sua página. Essa descrição será exibida abaixo do título ao buscar por sua página em um browser. -->

CSS

O CSS desempenha o papel da estilização do conteúdo de uma página, contribuindo significativamente para a sua apresentação visual. Embora seja possível incluir estilos diretamente nas tags HTML usando o atributo 'style', essa prática não é recomendada de acordo com as boas práticas de programação. Em vez disso, é aconselhável manter o CSS em um documento separado. Isso promove códigos mais limpos, legíveis e bem organizados, facilitando a manutenção e a compreensão do código.

  1. <body style=”color: white”></body> <!-- Não utilize dessa maneira! -->
  3. <link rel=”stylesheet” href=”enderecoDoDocumentoCSS”> <!-- Importe o documento CSS para o HTML com essa tag. -->
  5. <body class=”classe” id=”id”></body> <!-- Importe os estilos criados no documento CSS utilizando class ou id. -->

Importação do CSS e JS

De acordo com diversos estudos, é considerado uma prática recomendada importar o CSS utilizando a tag <link> dentro da seção <head> do documento HTML. Essa abordagem tende a melhorar a performance e a renderização da página, uma vez que o documento CSS é processado simultaneamente com os componentes do HTML.

Quanto ao JavaScript (JS), é aconselhável importá-lo utilizando a tag <script> antes do fechamento da tag </body>. Essa prática também contribui para a melhoria da performance na renderização, uma vez que a página realizará o processamento do documento JS somente após a exibição dos elementos HTML.

  1. <head>
  2.     <link rel="stylesheet" href="enderecoDoDocumento.css">
  3. </head>
  4. <body>
  5.     <!-- Conteúdo -->
  6.     <script src="enderecoDoDocumento.js"></script>
  7. </body>

Atributos de Acessibilidade

É fundamental fazer uso dos atributos de acessibilidade disponíveis no HTML, pois eles auxiliam programas externos na identificação de conteúdos em nossas páginas, possibilitam uma melhor utilização de atalhos, como por exemplo, a tecla TAB, e permitem que pessoas com necessidades especiais desfrutem do conteúdo e das funcionalidades da página.

Aqui estão alguns exemplos de atributos:

alt: Este atributo destina-se a fornecer uma descrição ao elemento HTML em que é utilizado. Dessa forma, se o elemento em questão não for renderizado, a descrição estará presente no seu lugar. Outro benefício muito importante é a descrição do conteúdo da página para pessoas com deficiência visual, já que os leitores de tela não conseguem interpretar elementos como imagens ou gifs, mas podem ler as descrições.

title: Este atributo é comumente utilizado para fornecer informações de orientação sobre elementos HTML. Ele também é lido por leitores de tela.

tabindex: Este atributo determina a ordem de tabulação (foco) ao pressionar a tecla TAB. O tabindex é útil para definir a ordem desejada dos elementos em que o foco deve se deslocar.

  1. <img src="enderecoDaImagem.jpg" alt="Descrição da imagem"> <!-- Descrevendo uma imagem com o atributo alt -->
  3. <button onclick="acao()" title="Objetivo desse botão"></button> <!-- Informando o objetivo do botão com o atributo title -->
  5. <a href="enderecoDoLink.com" tabindex="1"></a> <!-- Definindo a prioridade do TAB para uma tag específica com o atributo tabindex -->

Escrita Adequada

É importante que o seu código seja escrito da forma mais eficiente possível. Recomenda-se utilizar letras minúsculas ao declarar tags, métodos, atributos, entre outros elementos. Evite espaços desnecessários e excesso de linhas em branco. Organize o código com uma indentação apropriada. Embora os comentários sejam ferramentas úteis para identificar e documentar diferentes partes do código, é importante usá-los com moderação. Comentários em excesso podem dificultar a leitura e compreensão do código.

  1. <!-- Código incorreto -->
  2. <BODY ID="id">
  3.     <header CLASS="classe1">
  4. <!-- Comentário muito extenso -->
  5. <!-- Comentário desnecessário -->
  7. <H1   ></H1>
  8.         <IMG src="esderecoDaImagem"     ALT="Descrição da imagem">
  9.     </header>
  10. </body>
  12. <!-- Código correto -->
  13. <body id="id">
  14.     <header class="classe1">
  16.         <!-- Comentário necessário -->
  17.         <h1></h1>
  18.         <img src="enderecoDaImagem" alt="Descrição da imagem">
  19.     </header>
  20. </body>

Organização de Formulários

Ao desenvolver em HTML, é comum lidar com uma variedade de formulários. Portanto, recomenda-se o uso frequente das tags <fieldset> e <legend>, que são responsáveis por agrupar e descrever o conteúdo do formulário. Além disso, é importante fazer uso das tags <input> e <label>, que trabalham em conjunto para identificar e organizar as áreas de interação com o usuário.

  1. <form action="acao1">
  2.     <fieldset>
  3.         <legend>Legenda do Formulário 1</legend>
  5.         <label for="inp1"></label>
  6.         <input type="text" name="inp1" id="inp1">
  7.         <label for="inp2"></label>
  8.         <input type="text" name="inp2" id="inp2">
  9.     </fieldset>
  10. </form>

Tags Semânticas

Uma página web é composta por várias seções distintas, como cabeçalho, rodapé, barra de navegação, formulários, entre outras. Para manter o código bem organizado, é recomendável utilizar as tags apropriadas para cada parte da página. Embora existam diversas tags disponíveis para a organização semântica, não é necessário conhecer todas elas; basta dominar as mais básicas para obter uma estrutura eficiente.

Compressão

Para otimizar o desempenho do seu site, é altamente recomendável comprimir o código HTML, tornando-o mais leve e, consequentemente, mais rápido. Você pode realizar essa compressão manualmente utilizando ferramentas online específicas ou de forma automatizada por meio de ferramentas como Gulp ou Webpack.

Validação

Após concluir o desenvolvimento do seu HTML, é essencial validar o código, verificando se ele está em conformidade com os padrões e buscando por alertas e erros para corrigi-los. Para facilitar essa tarefa e identificar possíveis problemas que passaram despercebidos, é recomendável utilizar a ferramenta Lighthouse do Google. Você pode acessá-la através do DevTools, pressionando o atalho 'F12'.
